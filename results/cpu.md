## 2026-05-23 — Q4_0 tiled GEMM (`perf/cpu-performance`)

**Change:** Q4_0 prefill previously used a row-by-row 2-row GEMV loop (no tiled GEMM). Implemented `q4_gemm_tiled`: per-thread expand Q4 nibbles to int8, then reuse the existing 4×4 AVX-512 VNNI microkernel (`gemm_q8_tiled`). ## 2026-05-23 — GQA-aware attention prefill (`perf/cpu-performance`)

**Change:** Prefill attention was reading K and V from layout `[kv_len, num_kv_heads, head_dim]`, causing a stride of `num_kv_heads * head_dim` bytes between consecutive positions for the same KV head — non-unit stride prevents hardware prefetching. For GQA models (SmolLM2-360M has gqa_ratio=3), the same K/V head was also re-read separately for each of the 3 sharing query heads.

Fix: before parallel dispatch, transpose K and V to `[num_kv_heads, kv_len, head_dim]` (contiguous per KV head). Dispatch in `(seq_pos, kv_head)` units; each unit reads K once and V once for all `gqa_ratio` query heads together. Decode path (seq_len=1) is unchanged — transposition cost is not amortized there.

- **CPU:** AMD Ryzen AI 9 HX 370 w/ Radeon 890M (CPU in powersave mode — absolute numbers lower than governor-normal; back-to-back comparison is valid)
- **Threads:** 24 (all cores)
- **Decode tokens:** 256 | **Prefill tokens:** 512 (8-token warm-up prompt)
- **infernum branch:** `perf/cpu-performance`

### Prefill throughput improvement (back-to-back, same session)

| Workload | Before | After | Δ |
| -------- | -----: | ----: | -: |
| SmolLM2-360M Q8_0 prefill | 720 | 764 | **+6.1%** |
| SmolLM2-360M Q4_0 prefill | 679 | 773 | **+13.9%** |
| Gemma 2-2b Q8_0 prefill | 122 | 129 | **+5.3%** |
| SmolLM2-360M Q8_0 decode | 108 | 107 | −1% (noise) |
| SmolLM2-360M Q4_0 decode | 145 | 141 | −3% (noise) |

### Notes

- Decode is unaffected (transposition skipped for seq_len=1).
- The Q4_0 prefill gain (+13.9%) is larger than Q8_0 (+6.1%) because Q4_0 has more compute per GEMM step, so the relative cost of attention memory bandwidth was higher before the fix.
- Cumulative prefill vs llama.cpp: Q8_0 ~0.82×, Q4_0 ~0.72×, Gemma Q8_0 ~0.74× (extrapolated from % gains applied to governor-normal baseline).
